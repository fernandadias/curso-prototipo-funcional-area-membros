Na próxima aula você vai montar seu cockpit e instalar um **agente de código**,
o copiloto que vai construir junto com você. Mas tem mais de um no mercado, e
eles trocam de nome, preço e "melhor do momento" a cada poucos meses. Antes de
instalar, vale conhecer o terreno e fazer uma escolha tranquila. Spoiler: **o
curso vai usar o Claude Code** (a instalação começa na próxima aula), mas *Codex
e outros funcionam igual*. O que você aprende a fazer com um, faz com qualquer
um.

<Callout tipo="dica">
  **A ideia-âncora de toda a metodologia:** o processo de prototipação funcional

</Callout>

## O que é um "agente de código"

É o copiloto que mora no terminal (ou dentro do editor) e edita os arquivos do
projeto a partir do que você pede em linguagem natural (vamos usar o português
aqui). Você vai instalar o seu na próxima aula. **Claude Code** (da Anthropic) e
**Codex** (da OpenAI) são dois copilotos desse tipo. Ambos:

- Leem e editam os arquivos da "casa do projeto";
- Rodam comandos por você (instalar, iniciar o servidor, etc.);
- Mostram um **diff** (o "antes e depois") pra você aprovar;
- Mantêm o contexto da conversa enquanto você itera.

## Lado a lado

**🟫 Claude Code (Anthropic)**

- Roda no terminal e tem extensões pro VS Code/Cursor e um app.
- Forte em tarefas longas e em "ler o projeto inteiro" antes de agir.
- É o que o curso vai usar: você instala na próxima aula.

**🟦 Codex (OpenAI)**

- Também roda no terminal e tem integração com editor e nuvem.
- Forte em iteração rápida e em rodar tarefas em paralelo.
- Fluxo de aprovação e leitura de diff bem parecido.

As diferenças reais (quem está "melhor" hoje, preço, limites) **mudam o tempo
todo**. Por isso não vou te dar um veredito que vai expirar. Mas vou expor para
você como foi a minha avaliação para definir o Claude Code como base do curso.

## Decida o seu agente de código

Esta aula é de decisão, não de construção. Ainda não vamos instalar nada. O
objetivo é você chegar na próxima aula já sabendo qual agente vai usar.

1. **Liste o que você tem acesso hoje.** Sua empresa paga algum? Tem teste
   grátis? Anote as opções reais (não as ideais).
2. **Confira a disponibilidade do Claude Code.** Caso queira acompanhar
   exatamente o curso, o Claude Code é uma possibilidade para você/sua empresa?
3. **Escolha um pra começar.** A recomendação do curso é o Claude Code (os
   exemplos seguem nele), mas se você só tem Codex, bora com ele: os passos são
   os mesmos.
4. **Comprometa-se por algumas semanas.** Ficar numa só ferramenta no começo dá
   fluência. Trocar toda hora atrapalha o aprendizado.

</Callout>
